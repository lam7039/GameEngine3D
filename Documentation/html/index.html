Development Setup (Windows)

Requires: Visual Studio 2015 or higher, DirectX 9.0c, CMake and RapidXml

    • If using Git: Clone GameEngine3D to the same parent directory as ExecutingProject.
    • If using Copy: Go to Project
  2. Depending on your visual studio version run Project/VisualStudio(14/15)_x64_(C/D).bat (or x86 depending on your CMake installation) and open the solution in the newly created folder.
    • If not a single bat file works, then copy the bat with the correct visual studio and cmake version, then change the path to your cmake installation in a file editor
  3. Open the project properties. Under "Debugging", make sure the working directory is: $(ProjectDir)../../Development
  4. Open the project properties. Under "Linker -> System", make sure the subsystem is: Console (/SUBSYSTEM:CONSOLE)
Relevant links:
  • https://directx-sdk.soft32.com/
  • https://cmake.org/download/
  • https://sourceforge.net/projects/rapidxml/files/latest/download
  • https://github.com/lamram/GameEngine3D
  • https://github.com/lamram/ExecutingProject

Creating a level file with XML

To add assets, encapsulate an Asset with the Assets tag. Then give it a Type of Model, Skybox or Terrain. Also give an AssetName so an entity can use the asset. If the type of an asset is a Terrain, then you can add a HeightmapPath to the Asset. Finally give it an AssetPath, a path to the asset you want to load in. Example:
<Assets>
 <Asset>
  <Type>Model</Type>
  <AssetName>MyAmazingAssetName</AssetName>
  <AssetPath>Assets\\MyAmazingAsset.x</AssetPath>
 </Asset>
 <Asset>
  <Type>Terrain</Type>
  <AssetName>MyLessAmazingTerrain</AssetName>
  <AssetPath>Assets\\MyLessAmazingTexture.jpg</AssetPath>
  <HeightmapPath>Assets\\MyHeightmap.bmp</HeightmapPath>
 </Asset>
</Assets>
Adding scenes and entities in the scenes is a similar process, encapsulate Scene with the Scenes tag, you can give a name to a scene with the Name tag and add an entity with the Entity tag. In the entity tag you have to assign a Type, which are Model, Terrain, Skybox or Camera. You can add an asset that the entity is gonna use with the AssetName tag. you put the name of the added asset in. Other than that you can add a Position, Scale and Rotation which each encapsulates the x, y and z tags. Example:
<Scenes>
 <Scene>
  <Name>FirstScene</Name>
  <Entity>
   <Type>Model</Type>
   <AssetName>MyAmazingAssetName</AssetName>
   <Position>
    <x>0.0</x>
    <y>0.0</y>
    <z>10.0</z>
   </Position>
   <Rotation>
    <x>0.0</x>
    <y>0.0</y>
    <z>1.570796</z>
   </Rotation>
  </Entity>
 </Scene>
 <Scene>
  <Name>SecondScene</Name>
  <Entity>
   <Type>Model</Type>
   <AssetName>MyAmazingAssetName</AssetName>
   <Position>
    <x>0.0</x>
    <y>0.0</y>
    <z>10.0</z>
   </Position>
   <Rotation>
    <x>0.0</x>
    <y>0.0</y>
    <z>1.570796</z>
   </Rotation>
  </Entity>
  <Entity>
   <Type>Terrain</Type>
   <AssetName>MyLessAmazingTerrain</AssetName>
   <Position>
    <x>-5.0</x>
    <y>0.0</y>
    <z>10.0</z>
   </Position>
   <Scale>
    <x>2.0</x>
    <y>2.0</y>
    <z>2.0</z>
   </Scale>
  </Entity>
  <Entity>
   <Type>Camera</Type>
  </Entity>
 </Scene>
</Scenes>
After having created a level file you can load it with the SceneLoader like: "se::SceneLoader loader(renderer "Assets\\level.xml", input)".
